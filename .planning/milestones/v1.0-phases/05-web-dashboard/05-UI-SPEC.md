# Phase 05 — UI Design Contract

> Visual and interaction contract for the web dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Rust project -- no JS tooling) |
| Preset | not applicable |
| Component library | none -- pure HTML served via askama compile-time templates |
| Icon library | none -- text labels and Unicode symbols only |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` |

CSS is embedded in a `<style>` block within the base layout template. No external CSS files, no build step, no Tailwind. All styles are authored as plain CSS within askama templates.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between adjacent inline elements |
| sm | 8px | Compact element spacing, input internal padding (vertical) |
| md | 16px | Default element spacing, input internal padding (horizontal), card padding |
| lg | 24px | Section padding, nav bar padding, card body padding |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Page top/bottom margins |
| 3xl | 64px | Not used in this phase |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height | Element |
|------|------|--------|-------------|---------|
| Body | 16px | 400 (regular) | 1.5 | `p`, `td`, `li`, result text |
| Label | 14px | 600 (semibold) | 1.4 | `label`, `th`, nav links, metadata labels |
| Heading | 20px | 600 (semibold) | 1.2 | `h2` (page titles) |
| Display | 28px | 600 (semibold) | 1.2 | `h1` (site title in nav: "local-index") |

Font weights used: 400, 600. No other weights.

Monospace for code/paths: `"SF Mono", "Fira Code", "Fira Mono", Menlo, Consolas, "Courier New", monospace` at 14px/400/1.5. Used for: file paths, heading breadcrumbs, chunk text snippets, config values on settings page.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Page background, input backgrounds |
| Secondary (30%) | `#f5f5f5` | Nav bar background, table header row, result card backgrounds, settings key-value rows (alternating) |
| Accent (10%) | `#1a73e8` | Search button, active nav link underline, score badge text |
| Destructive | not used | No destructive actions in this phase (all views are read-only) |

Additional semantic colors:

| Role | Value | Usage |
|------|-------|-------|
| Text primary | `#1a1a1a` | Body text, headings |
| Text secondary | `#5f6368` | Metadata labels, timestamps, score values |
| Border | `#dadce0` | Input borders, table borders, card borders, nav bottom border |
| Accent hover | `#1557b0` | Search button hover state |
| Accent text on white | `#1a73e8` | Hyperlinks (file paths in results that link to index view) |

| Warning | `#ea8600` | Status page: "stale files" count when > 0 |

Accent reserved for: search submit button background, active nav link bottom-border, link text color. Never used for backgrounds of content areas.

## Layout Contract

### Base Layout (all pages)

```
+----------------------------------------------------------+
| nav: [local-index]   Search  Index  Status  Settings     |
+----------------------------------------------------------+
| <main> (max-width: 720px, centered, padding: 24px)       |
|                                                           |
|   [page content]                                          |
|                                                           |
+----------------------------------------------------------+
```

- Nav bar: full-width, `#f5f5f5` background, 1px bottom border `#dadce0`, height 48px, items vertically centered.
- Nav brand "local-index" is left-aligned, 20px semibold, monospace font.
- Nav links are right-aligned, 14px semibold, 16px horizontal gap between links.
- Active nav link has a 2px bottom border in accent color.
- Main content: `max-width: 720px`, `margin: 0 auto`, `padding: 48px 24px 48px 24px`.

### Search Page (`/` and `/search`)

```
+----------------------------------------------------------+
| [text input, full width              ] [mode v] [Search Notes]|
+----------------------------------------------------------+
| Showing N results for "query" (hybrid mode)               |
+----------------------------------------------------------+
| Result 1                                                  |
| filepath.md > Heading > Subheading          score: 0.82  |
| Chunk text preview, first 200 characters of the chunk... |
+----------------------------------------------------------+
| Result 2                                                  |
| ...                                                       |
+----------------------------------------------------------+
```

- Search form: single row. Text input takes remaining width. Mode `<select>` (hybrid/semantic/fts) is 120px wide. Submit button "Search Notes" is accent-colored.
- Results summary line: 14px, secondary text color, below form with 16px gap.
- Each result card: 1px border `#dadce0`, border-radius 4px, padding 16px, margin-bottom 16px.
- Result card line 1: file path in monospace 14px as a plain text label + breadcrumb, right-aligned score badge.
- Result card line 2: chunk text preview, body style (16px/400), truncated to 3 lines with `overflow: hidden`.
- Score badge: accent text color, 14px semibold, no background.

### Index Browser (`/index`)

```
+----------------------------------------------------------+
| Index Browser                                             |
+----------------------------------------------------------+
| File                          | Chunks | Last Indexed     |
|-------------------------------|--------|------------------|
| notes/project.md              | 12     | 2026-04-10 14:32 |
| notes/ideas.md                | 5      | 2026-04-10 14:32 |
+----------------------------------------------------------+
| Showing 42 files, 284 total chunks                        |
+----------------------------------------------------------+
```

- HTML `<table>` with full width.
- Table header: `#f5f5f5` background, 14px semibold labels.
- File path column: monospace 14px.
- Chunks column: right-aligned, body style.
- Last Indexed column: secondary text color, body style.
- Summary row below table: 14px, secondary text color.

### Status Page (`/status`)

```
+----------------------------------------------------------+
| Index Status                                              |
+----------------------------------------------------------+
| Total Files          | 42                                 |
| Total Chunks         | 284                                |
| Last Index Time      | 2026-04-10 14:32:07 UTC            |
| Pending Queue        | 0                                  |
| Stale Files          | 0                                  |
+----------------------------------------------------------+
| Embedding Stats                                           |
+----------------------------------------------------------+
| Model                | voyage-3.5                          |
| Total Embeddings     | 284                                |
| Dimensions           | 1024                               |
+----------------------------------------------------------+
```

- Key-value layout using a definition list (`<dl>`) or two-column table.
- Alternating row backgrounds: white / `#f5f5f5`.
- Section headings: 20px semibold, 24px top margin between sections.
- Values in monospace where numeric or technical.
- "Stale Files" value: warning color (`#ea8600`) when > 0, otherwise primary text.
- "Pending Queue" value: warning color when > 0.

### Settings Page (`/settings`)

```
+----------------------------------------------------------+
| Settings                                                  |
+----------------------------------------------------------+
| Data Directory       | ~/.local-index                     |
| Bind Address         | 127.0.0.1:3000                     |
| Embedding Provider   | voyage                             |
| Credential Source    | VOYAGE_API_KEY env var              |
| Log Level            | info                               |
+----------------------------------------------------------+
```

- Same key-value layout as status page.
- All values in monospace.
- Read-only, no form elements.
- Single section, no sub-sections needed.

## Interaction Contract

### Form Behavior

| Interaction | Mechanism | Detail |
|-------------|-----------|--------|
| Search submit | `<form method="GET" action="/search">` | Full page reload, query params in URL |
| Mode selection | `<select name="mode">` | Options: hybrid (default), semantic, fts |
| Result limit | Not exposed in UI | Server uses default limit of 20 |
| Enter to search | Native form behavior | No JS needed |
| Browser back/forward | URL-based state | Query preserved in URL params |

### States

| State | Where | What User Sees |
|-------|-------|----------------|
| Empty search (no query) | `/` | Search form only, no results section |
| Search with results | `/search?q=foo` | Form pre-filled + results list |
| Search with zero results | `/search?q=foo` | Form pre-filled + "No matching notes found" message |
| Empty index | `/index` | Empty state message (see Copywriting) |
| Empty status (no data) | `/status` | All values show "0" or "never" |
| Server error | Any route | 500 page with error message |

### Navigation

- All nav links are standard `<a>` tags.
- Active link is determined server-side by comparing the current route.
- No JavaScript for navigation state.

## Component Inventory

These are the askama templates needed. Each maps to one `.html` file in `templates/`.

| Template | Purpose | Includes |
|----------|---------|----------|
| `base.html` | Layout shell: doctype, head, nav, main wrapper, embedded CSS | All pages extend this |
| `search.html` | Search form + optional results list | Extends `base.html` |
| `index.html` | File table + empty state | Extends `base.html` |
| `status.html` | Key-value status + embedding stats | Extends `base.html` |
| `settings.html` | Key-value config display | Extends `base.html` |
| `error.html` | 500 error page | Extends `base.html` |

No partial/fragment templates needed in v1. The embedded CSS lives in `base.html` inside a `<style>` tag.

## Accessibility Baseline

- All form inputs have associated `<label>` elements.
- Table headers use `<th scope="col">`.
- Color contrast ratio: `#1a1a1a` on `#ffffff` = 17.4:1 (AAA). `#5f6368` on `#ffffff` = 5.9:1 (AA). `#1a73e8` on `#ffffff` = 4.6:1 (AA for large text / UI components).
- Focus styles: browser default outline (not suppressed).
- Semantic HTML: `<nav>`, `<main>`, `<table>`, `<form>`, `<h1>`, `<h2>`.
- No ARIA needed -- content is static HTML with native semantics.
