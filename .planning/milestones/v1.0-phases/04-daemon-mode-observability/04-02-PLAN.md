<objective>
Implement the `status` CLI command that reports index statistics by querying LanceDB directly.

Purpose: CLI-04 requires operators to check index health without running the daemon. The status command opens LanceDB, counts rows and distinct files, and reports the results. This is independent of the daemon/metrics infrastructure and can be built in parallel.

Output: Working `local-index status` command that outputs total chunks, total files, last index time, queue depth (0 when daemon is not running), stale file count (0 for now -- no mtime tracking yet), in both JSON and pretty formats.

Replace the `cli::Command::Status` arm in main.rs with a full implementation:

```rust
cli::Command::Status => {
    tracing::info!("status command invoked");

    // Resolve data directory (same pattern as search command)
    let data_dir = cli.data_dir.clone()
        .unwrap_or_else(|| std::env::current_dir().unwrap().join(".local-index"));

    if !data_dir.exists() {
        if is_tty {
            println!("No index found at '{}'.", data_dir.display());
            println!("Run `local-index index <path>` first, or specify --data-dir.");
        } else {
            let status = serde_json::json!({
                "error": format!("No index found at '{}'", data_dir.display())
            });
            println!("{}", serde_json::to_string(&status)?);
        }
        return Ok(());
    }

    let db_path = data_dir.to_string_lossy().to_string();
    let store = local_index::pipeline::store::ChunkStore::open(&db_path).await?;

    let total_chunks = store.count_total_chunks().await.unwrap_or(0);
    let total_files = store.count_distinct_files().await.unwrap_or(0);

    // Determine TTY vs pipe output
    let is_tty = std::io::stdout().is_terminal();

    if is_tty {
        println!("Index Status");
        println!("============");
        println!("Total chunks:     {}", total_chunks);
        println!("Total files:      {}", total_files);
        println!("Last index time:  unknown");
        println!("Queue depth:      0 (daemon not running)");
        println!("Stale files:      0");
        println!("Data directory:   {}", data_dir.display());
    } else {
        let status = serde_json::json!({
            "total_chunks": total_chunks,
            "total_files": total_files,
            "last_index_time": null,
            "queue_depth": 0,
            "queue_depth_note": "daemon not running",
            "stale_files": 0,
            "data_dir": data_dir.to_string_lossy()
        });
        println!("{}", serde_json::to_string(&status)?);
    }
}
```

Move the `let is_tty = std::io::stdout().is_terminal();` declaration that currently exists inside the Index command arm to be accessible to the Status arm as well (or just create a new one in the Status arm since the existing one is scoped inside the Index block).

Note: `last_index_time` is `null`/`unknown` for now because the LanceDB schema has no `indexed_at` column. This will be populated when daemon mode adds per-file timestamps. `stale_files` is 0 for the same reason. These are accurate representations of current capability, not placeholders.
