  <action>
    Replace the `Commands::Index` arm in `main()` with the full embed+store pipeline. The main function must become async (`#[tokio::main]`).

    **Step 1: Make main async.**
    Change `fn main() -> Result<()>` to `#[tokio::main] async fn main() -> Result<()>`. All `.await` calls now work.

    **Step 2: Resolve data directory.**
    ```rust
    let data_dir = cli.data_dir
        .clone()
        .unwrap_or_else(|| vault_path.join(".local-index"));
    let db_path = data_dir.to_string_lossy().to_string();
    ```

    ```rust
    let api_key = local_index::credentials::resolve_voyage_key()?;
    ```
    This fails with clear error if VOYAGE_API_KEY is not set.

    **Step 4: Create embedder and store.**
    ```rust
    let embedder = VoyageEmbedder::new(api_key);
    let store = ChunkStore::open(&db_path).await?;
    ```

    **Step 5: Check model consistency (per INDX-06).**
    ```rust
    let needs_full_reindex = store.check_model_consistency(embedder.model_id(), *force_reindex).await?;
    if needs_full_reindex {
        tracing::warn!("model changed, clearing all existing data for full re-index");
        store.clear_all().await?;
    }
    ```

    **Step 6: Walk and chunk files** (same as current, but collect into per-file groups).

    **Step 7: Create progress reporting (per D-06, D-07, D-08).**
    Detect TTY via `std::io::stdout().is_terminal()` and store in a `let is_tty: bool`.

    For **TTY mode** (per D-07): create an indicatif ProgressBar:
    ```rust
    use std::io::IsTerminal;
    use indicatif::{ProgressBar, ProgressStyle};

    let is_tty = std::io::stdout().is_terminal();
    let pb = if is_tty {
        let pb = ProgressBar::new(files.len() as u64);
        pb.set_style(
            ProgressStyle::default_bar()
                .template("{spinner:.green} [{bar:40.cyan/blue}] {pos}/{len} files ({msg})")
                .unwrap()
                .progress_chars("=>-")
        );
        Some(pb)
    } else {
        None
    };
    ```

    For **non-TTY mode** (per D-08): use `eprintln!` to emit progress lines to stderr after each file is processed. This is critical — D-08 requires "progress lines to stderr" so that pipe consumers (e.g., Claude Code) can monitor progress while the JSON summary goes to stdout. Example per-file line:
    ```rust
    if !is_tty {
        eprintln!("[{}/{}] {} — {} chunks embedded, {} skipped",
            file_index + 1, total_files, relative_path_str,
            file_chunks_embedded, file_chunks_skipped);
    }
    ```

    **Step 8: Process each file with incremental skip logic.**
    For each file:
    1. Read file content (existing pattern — warn and continue on error)
    2. Chunk with `chunk_markdown()` (existing)
    3. Compute content hashes for all chunks via `compute_content_hash()`
    4. Get existing hashes from store: `store.get_hashes_for_file(relative_path_str).await`
    5. Filter: only chunks whose hash is NOT in existing hashes need embedding
    6. If no chunks need embedding: increment skip counter, update progress (bar or eprintln!), continue
    7. If chunks need embedding:
       a. Collect chunk bodies as `Vec<String>`
       b. Call `embedder.embed(&texts).await` (handles batching and retry internally)
       c. On success: `store.delete_chunks_for_file(relative_path_str).await` then `store.store_chunks(all_file_chunks, all_embeddings, all_hashes, embedder.model_id()).await`
       d. On error: `tracing::warn!` with file path and error, increment error counter, continue (per-file graceful error handling)
    8. Update progress:
       - TTY: `pb.as_ref().unwrap().set_message(format!(...))` and `pb.as_ref().unwrap().inc(1)`
       - Non-TTY: `eprintln!(...)` with per-file progress line to stderr

    **Step 9: Finish and output summary.**
    ```rust
    if let Some(pb) = &pb {
        pb.finish_and_clear();
    }

    if is_tty {
        // Per D-07: human-readable summary line
        println!("Indexed {} files | {} chunks embedded | {} skipped | {} errors",
            file_count, chunks_embedded, chunks_skipped, error_count);
    } else {
        // Per D-08: JSON summary to stdout
        let summary = serde_json::json!({
            "files_indexed": file_count,
            "chunks_embedded": chunks_embedded,
            "chunks_skipped": chunks_skipped,
            "errors": error_count
        });
        println!("{}", serde_json::to_string(&summary)?);
    }
    ```

    **Important details:**
    - Remove the old JSONL output code entirely
    - Keep the existing `serde_json` import (already in Cargo.toml)
    - Import: `use local_index::credentials::resolve_voyage_key;`
    - Import: `use local_index::pipeline::embedder::VoyageEmbedder;`
    - Import: `use local_index::pipeline::store::{ChunkStore, compute_content_hash};`
    - Import: `use indicatif::{ProgressBar, ProgressStyle};`
    - Import: `use std::io::IsTerminal;`
    - The entire index arm should be wrapped in the tokio async runtime
    - For the "skip" logic: compare the SET of computed hashes with the SET of stored hashes. If they are identical (same hashes, same count), the file is unchanged. Otherwise, re-embed the **whole file** (delete old + insert new). This is simpler than per-chunk diffing and correct because: (a) chunks now overlap — a single content change invalidates many overlapping chunks, and (b) we process entire files at a time. Do NOT try to diff individual chunks.
    - **Chunk count per file is variable:** The smart chunker (Plan 01 Task 1) produces size-based overlapping chunks. A short file → 1 chunk; a long file → N chunks with CHUNK_OVERLAP_CHARS overlap. The index command iterates whatever `chunk_markdown()` returns without assuming count.
