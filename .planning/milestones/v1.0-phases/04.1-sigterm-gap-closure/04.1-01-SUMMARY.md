## Accomplishments

- Replaced single `ctrl_c()` await with `tokio::select!` that races SIGINT and SIGTERM, so process managers (systemd, launchctl, Docker) can trigger graceful shutdown via `kill <pid>`
- Added two unit tests (`test_sigterm_cancels_token`, `test_sigint_still_cancels_token`) using `libc::kill` to deliver signals in-process, verifying cancellation within 500ms
- Added `libc = "0.2"` to dev-dependencies for test signal delivery

## Files Created/Modified

- `Cargo.toml` - Added `libc = "0.2"` to dev-dependencies
- `Cargo.lock` - Updated after libc addition

## Decisions Made

- No `cfg(unix)` gate added — CLAUDE.md confirms macOS (primary) and Linux (secondary) are the only targets; the gate would add noise with no benefit
- Used `libc::kill` in tests rather than the `nix` crate — `libc` is already available transitively, keeping dev-dependency count low
- Second SIGTERM after graceful shutdown start is intentionally ignored — the process will drain tasks and exit naturally; only a second SIGINT triggers `std::process::exit(1)`
