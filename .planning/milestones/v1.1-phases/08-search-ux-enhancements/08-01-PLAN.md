2. Create `src/web/highlight.rs` exporting `fn highlight_query_terms(preview: &str, query: &str) -> String`:
   - Split `query` on ASCII whitespace into non-empty terms; `regex::escape` each term; build one case-insensitive regex with word boundaries, e.g. `(?i)(?:\b(term1)\b|\b(term2)\b)`.
   - Find non-overlapping leftmost matches in `preview`; between matches append HTML-escaped slices; for each matched substring append `format!(\"<mark>{}</mark>\", encode(match_slice))` where `encode` is the text HTML entity encoder from the chosen crate (only the **match text** is encoded, not the `mark` delimiters).
   - Empty query: return fully escaped preview with zero `<mark>` tags.
3. Add `mod highlight; pub use highlight::highlight_query_terms;` (or `pub(crate) fn`) in `src/web/mod.rs` as appropriate for existing module layout.

   - Multi-word `query` "foo bar" highlights both in text "Foo is bar"
   - Case-insensitive match
   - Word boundary: query `bar` does not highlight inside `foobar`
   - XSS attempt: query `&lt;script&gt;` and preview containing user-like text — output must contain **escaped** entities (e.g. `&amp;lt;`) and must **not** contain raw `&lt;script&gt;`

  <action>
**SearchParams (`src/web/handlers.rs`):**
- Add `pub rerank: Option<bool>` with `#[serde(default)]`.
- Keep `no_rerank: Option<bool>`.

**Rerank resolution:**
- Implement exactly: `let rerank = state.anthropic_reranker.is_some() && !params.no_rerank.unwrap_or(false) && params.rerank != Some(false);`
- Truth table to satisfy in tests or comments: (a) no reranker ⇒ always `false`; (b) `no_rerank=true` ⇒ `false`; (c) `rerank=false` ⇒ `false`; (d) reranker + `rerank=true` + no `no_rerank` ⇒ `true`; (e) reranker + both absent ⇒ `true` (default-on when checkbox not unchecked).
- Template repopulation: `rerank_checked = state.anthropic_reranker.is_some() && !params.no_rerank.unwrap_or(false) && params.rerank != Some(false)` after a search; on empty query page, default `rerank_checked` to `anthropic_reranker.is_some()`.

**SearchTemplate new fields:**
- `rerank_available: bool`
- `rerank_checked: bool` (form repopulation)
- `rerank_applied: bool` (true when `rerank && reranker_available` after search)
- Preserve `active_nav`.

**SearchResultView:**
- Replace raw snippet usage with `pub chunk_html: askama::filters::Safe<String>` (import `askama::filters::Safe`).
- When mapping results, set `chunk_html: Safe(highlight_query_terms(&preview, &query))`.

**Empty query branch:** pass sensible defaults for new bools (checkbox checked if reranker available).

**`templates/search.html`:**
- Checkbox: `<input type="checkbox" name="rerank" value="true" ...>` only when `rerank_available`; set HTML `checked` when `rerank_checked`.
- Include `<input type="hidden" name="no_rerank" value="true" {% if rerank_checked %}disabled{% endif %}>` when `rerank_available` (Askama conditional). HTML5 **does not submit disabled controls**, so: **checked** ⇒ only `rerank=true` is sent; **unchecked** ⇒ only `no_rerank=true` is sent — no duplicate keys, **no JavaScript** required.
- `disabled` + `title="..."` + link `<a href="/settings" class="text-secondary">Settings</a>` beside label when `!rerank_available`.
- Results summary: if `rerank_applied`, append a muted `(reranked)` suffix once (e.g. wrap in `<span class="text-secondary">(reranked)</span>` inside the existing summary `<p>`) per D-03/D-04.
- `.result-body`: output `{{ result.chunk_html }}` (must not use `{{ result.chunk_text }}` for body anymore; remove `chunk_text` from view if unused, or keep for debug-only — prefer single field).

Run `cargo test` and `cargo check`.
