  <action>
    Implement `src/pipeline/chunker.rs` with two public functions:

    **`pub fn chunk_markdown(content: &str, file_path: &Path) -> Result<ChunkedFile, LocalIndexError>`**

    Algorithm:
    1. Set up pulldown-cmark options: `Options::ENABLE_YAML_STYLE_METADATA_BLOCKS | Options::ENABLE_HEADING_ATTRIBUTES`
    2. Create parser with `Parser::new_ext(content, options).into_offset_iter()`
    3. Track state:
       - `heading_stack: Vec<(HeadingLevel, String)>` for breadcrumb hierarchy
       - `current_heading_text: String` for accumulating heading text across multiple events
       - `in_heading: bool` flag
       - `in_metadata: bool` flag
       - `yaml_text: String` for frontmatter accumulation
       - `current_body: String` for chunk body accumulation
       - `current_line_start: usize` (1-based line number)
       - `chunks: Vec<Chunk>`
       - `frontmatter: Frontmatter`
    4. Event processing:
       - `Event::Start(Tag::MetadataBlock(MetadataBlockKind::YamlStyle))`: set in_metadata = true
       - `Event::Text(text)` when in_metadata: append to yaml_text
       - `Event::End(TagEnd::MetadataBlock(_))`: parse yaml_text with `serde_yml::from_str::<Frontmatter>()`, log warning on parse failure, set in_metadata = false, record frontmatter end offset for line counting
       - `Event::Start(Tag::Heading { level, .. })`: set in_heading = true, clear current_heading_text
       - `Event::Text(text)` / `Event::Code(code)` when in_heading: append to current_heading_text
       - `Event::End(TagEnd::Heading(_))`:
         a. Finalize previous chunk if current_body is non-empty (trim whitespace; skip if only whitespace)
         b. Update heading stack: pop entries with level >= current level, push new (level, text)
         c. Build breadcrumb string by joining stack entries with " > ", each prefixed with "#" repeated by level (e.g., `# A > ## B`)
         d. Reset current_body, set in_heading = false, update current_line_start
       - `Event::Text(text)` / `Event::Code(code)` / `Event::SoftBreak` / `Event::HardBreak` when not in_heading and not in_metadata: append to current_body (SoftBreak/HardBreak as "\n")
    5. After all events: finalize last chunk if current_body is non-empty
    6. For line number tracking: count newlines in content[..offset] to derive line numbers from byte offsets. Helper function: `fn byte_offset_to_line(content: &str, offset: usize) -> usize` that counts '\n' characters up to offset and adds 1 (1-based).
    7. Return `ChunkedFile { file_path: file_path.to_path_buf(), frontmatter, chunks }`

    **Helper functions:**
    - `fn update_heading_stack(stack: &mut Vec<(HeadingLevel, String)>, level: HeadingLevel, text: String)`: pop while last >= level, push new
    - `fn breadcrumb(stack: &[(HeadingLevel, String)]) -> String`: join with " > ", each entry formatted as `{"#" * level} {text}`
    - `fn byte_offset_to_line(content: &str, offset: usize) -> usize`: count '\n' up to offset, add 1

    **Frontmatter handling:** When `serde_yml::from_str` fails, log a warning with `tracing::warn!(error = %e, "failed to parse YAML frontmatter; treating as content")` and set frontmatter to `Frontmatter::default()`.

    - `test_empty_sections`: heading immediately followed by another heading
    - `test_malformed_frontmatter`: invalid YAML does not panic, defaults used
    - `test_deeply_nested_headings`: H1 through H6 breadcrumb
