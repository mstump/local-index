# Phase 1, Plan 1: Rust Project Setup and CLI Skeleton

## Objective

Initialize the Rust project with Cargo, set up the CLI skeleton using clap derive macros with all subcommands (index, daemon, search, status, serve), configure structured logging via tracing, and establish .env support via dotenvy. The binary should compile, run `--help`, and emit structured logs.

## Context

- @.planning/PROJECT.md - Project definition, constraints, tech stack
- @CLAUDE.md - Project conventions and technology choices

## Tasks

### Task 1: Initialize Cargo project
type="auto"

Create the Rust project with `cargo init`, set up Cargo.toml with initial dependencies needed for this plan: clap (with derive), tracing, tracing-subscriber, dotenvy, anyhow, thiserror.

**Done when:**
- `Cargo.toml` exists with all required dependencies
- `cargo check` passes

### Task 2: Define CLI structure with clap derive
type="auto"

Create the CLI argument parser using clap derive macros. Define all subcommands: index (takes path), daemon (takes path), search (takes query), status, serve. Add global flags: --log-level, --data-dir. Add .env loading via dotenvy before clap parses. Each subcommand should have its relevant flags as documented in requirements.

**Done when:**
- CLI struct defined with all subcommands
- `cargo run -- --help` shows all subcommands
- Each subcommand has `--help` with meaningful descriptions
- `.env` file is loaded before CLI parsing

### Task 3: Set up structured logging with tracing
type="auto"

Configure tracing-subscriber with EnvFilter for RUST_LOG support, connect --log-level CLI flag to the tracing subscriber, and add initial tracing instrumentation to main. Logging should be structured (not plain text).

**Done when:**
- `tracing_subscriber` initialized with env filter
- `--log-level` flag controls log verbosity
- `RUST_LOG` env var also works
- Running any subcommand produces structured log output

### Task 4: Add basic main dispatch and error handling
type="auto"

Wire up the main function to dispatch to subcommand stubs. Each subcommand should log that it was invoked and exit cleanly. Set up anyhow for error propagation. Ensure the binary compiles and all subcommands work as stubs.

**Done when:**
- `cargo build` succeeds
- `cargo run -- index /tmp` logs invocation and exits
- `cargo run -- search "test"` logs invocation and exits
- All subcommands are reachable
- Error handling uses anyhow::Result

## Verification

```bash
cargo build 2>&1
cargo run -- --help
cargo run -- index --help
cargo run -- search --help
cargo run -- daemon --help
cargo run -- status --help
cargo run -- serve --help
RUST_LOG=debug cargo run -- index /tmp 2>&1 | head -5
```

## Success Criteria

1. Binary compiles without warnings
2. `--help` shows all 5 subcommands with descriptions
3. Each subcommand has its own `--help` with relevant flags
4. Structured logging works via both `--log-level` and `RUST_LOG`
5. All subcommands execute as stubs and exit cleanly

## Output

- `Cargo.toml` - Project manifest with dependencies
- `src/main.rs` - CLI entry point with subcommand dispatch
- `src/cli.rs` - CLI argument definitions (clap derive)
