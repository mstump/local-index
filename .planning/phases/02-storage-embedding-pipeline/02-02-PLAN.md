    **src/pipeline/mod.rs** — Add `pub mod store;`

    **src/pipeline/store.rs** — Create with:

    1. **Constants:**
    ```rust
    pub const TABLE_NAME: &str = "chunks";
    pub const EMBEDDING_DIM: i32 = 1024;
    ```

    2. **compute_content_hash(chunk: &Chunk) -> String** (per D-09):
    - Create `sha2::Sha256` hasher
    - Update with `chunk.body.as_bytes()`
    - Update with `chunk.heading_breadcrumb.as_bytes()`
    - Update with `serde_json::to_string(&chunk.frontmatter).unwrap_or_default().as_bytes()`
    - Return `format!("{:x}", hasher.finalize())`

    3. **chunks_schema() -> Arc<Schema>** — Arrow schema with these 10 fields:
    - `chunk_id`: Utf8, not null
    - `file_path`: Utf8, not null
    - `heading_breadcrumb`: Utf8, not null
    - `body`: Utf8, not null
    - `line_start`: UInt32, not null
    - `line_end`: UInt32, not null
    - `frontmatter_json`: Utf8, nullable
    - `content_hash`: Utf8, not null
    - `embedding_model`: Utf8, not null
    - `vector`: FixedSizeList(Float32, EMBEDDING_DIM), not null

    4. **ChunkStore struct:**
    ```rust
    pub struct ChunkStore {
        db: lancedb::Connection,
        table: lancedb::Table,
    }
    ```

    5. **ChunkStore::open(db_path: &str) -> Result<Self, LocalIndexError>:**
    - `lancedb::connect(db_path).execute().await`
    - Try `db.open_table(TABLE_NAME).execute().await`
    - If table doesn't exist, `db.create_empty_table(TABLE_NAME, chunks_schema()).execute().await`
    - Return `ChunkStore { db, table }`

    6. **ChunkStore::store_chunks(chunks, embeddings, hashes, model_id) -> Result<(), LocalIndexError>:**
    - Takes `&[Chunk]`, `&[Vec<f32>]`, `&[String]` (hashes), `&str` (model_id)
    - Build Arrow arrays for each column:
      - chunk_id: `StringArray` from `uuid::Uuid::new_v4().to_string()` for each chunk
      - file_path: `StringArray` from `chunk.file_path.to_string_lossy()`
      - heading_breadcrumb: `StringArray`
      - body: `StringArray`
      - line_start: `UInt32Array` from `chunk.line_start as u32`
      - line_end: `UInt32Array` from `chunk.line_end as u32`
      - frontmatter_json: `StringArray` from `serde_json::to_string(&chunk.frontmatter).ok()`
      - content_hash: `StringArray` from hashes
      - embedding_model: `StringArray` (all same model_id)
      - vector: `FixedSizeListArray::from_iter_primitive::<Float32Type, _, _>`
    - Build `RecordBatch::try_new(chunks_schema(), columns)`
    - `self.table.add(vec![batch]).execute().await`

    7. **ChunkStore::get_hashes_for_file(file_path: &str) -> Result<Vec<String>, LocalIndexError>:**
    - `self.table.query().filter(format!("file_path = '{}'", file_path)).select(["content_hash"]).execute().await`
    - Collect RecordBatches via `futures::TryStreamExt::try_collect::<Vec<_>>()`
    - Extract content_hash strings from each batch's StringArray column
    - Return collected hashes

    8. **ChunkStore::delete_chunks_for_file(file_path: &str) -> Result<(), LocalIndexError>:**
    - `self.table.delete(&format!("file_path = '{}'", file_path)).await`

    9. **ChunkStore::check_model_consistency(model_id: &str, force_reindex: bool) -> Result<bool, LocalIndexError>:**
    - Query: `self.table.query().select(["embedding_model"]).limit(1).execute().await`
    - If no rows: return `Ok(false)` (no data, fresh DB — false means "no existing data to clear")
    - If rows exist: extract embedding_model from first row
    - If stored_model == model_id: return `Ok(false)` (matches, no action needed)
    - If stored_model != model_id AND force_reindex: return `Ok(true)` (true means "caller should clear and re-embed all")
    - If stored_model != model_id AND !force_reindex: return `Err(LocalIndexError::Database(format!("Embedding model mismatch: database contains '{}' but configured model is '{}'. Run with --force-reindex to re-embed all chunks.", stored_model, model_id)))`

    10. **ChunkStore::clear_all() -> Result<(), LocalIndexError>:**
    - `self.table.delete("true").await` (delete all rows for force-reindex)

    11. **Unit tests** (all use `tempfile::tempdir()` for DB path):
    - `test_open_creates_table`: Open store in empty tempdir, verify no panic
    - `test_store_and_retrieve_hashes`: Store 2 chunks, call get_hashes_for_file, verify 2 hashes returned
    - `test_get_hashes_unknown_file`: get_hashes_for_file on nonexistent file returns empty vec
    - `test_delete_chunks_for_file`: Store chunks for 2 files, delete one, verify only that file's chunks removed
    - `test_model_consistency_empty_db`: check_model_consistency on fresh DB returns Ok(false)
    - `test_model_consistency_match`: Store with "voyage-3.5", check "voyage-3.5" returns Ok(false)
    - `test_model_consistency_mismatch_no_force`: Store with "voyage-3.5", check "voyage-4" without force returns Err
    - `test_model_consistency_mismatch_force`: Store with "voyage-3.5", check "voyage-4" with force returns Ok(true)
    - `test_compute_hash_deterministic`: Same chunk produces same hash twice
    - `test_compute_hash_body_change`: Different body produces different hash
    - `test_compute_hash_breadcrumb_change`: Different heading_breadcrumb produces different hash
    - `test_compute_hash_frontmatter_change`: Different frontmatter produces different hash

    All LanceDB errors should be mapped to `LocalIndexError::Database(e.to_string())`.
