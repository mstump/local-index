    - `VoyageResponse { data: Vec<VoyageEmbedding>, model: String, usage: VoyageUsage }`
    - `VoyageEmbedding { embedding: Vec<f32>, index: usize }`
    - `VoyageUsage { total_tokens: u64 }`

    3. **VoyageEmbedder struct** (per D-03):
    ```rust
    pub struct VoyageEmbedder {
        client: reqwest::Client,
        api_key: String,
        model: String,
        dimensions: usize,
        base_url: String,  // default "https://api.voyageai.com" — configurable for tests
        batch_size: usize, // default 50
    }
    ```
    Constructor: `VoyageEmbedder::new(api_key: String) -> Self` with defaults: model = "voyage-3.5", dimensions = 1024, batch_size = 50, base_url = "https://api.voyageai.com".
    Also: `VoyageEmbedder::with_base_url(api_key: String, base_url: String) -> Self` for testing with wiremock.

    4. **Embedder trait impl for VoyageEmbedder**:
    - `embed()`: Split input texts into batches of `self.batch_size`. For each batch, call `embed_batch_with_retry()`. Concatenate all EmbeddingResults (merge embeddings vecs, sum total_tokens).
    - `model_id()`: return `&self.model`
    - `dimensions()`: return `self.dimensions`

    5. **embed_batch_with_retry()** private async method (per INDX-07):
    - Constants: MAX_RETRIES=5, BASE_DELAY_MS=500, MAX_DELAY_MS=30_000
    - Loop up to MAX_RETRIES attempts
    - POST to `{base_url}/v1/embeddings` with `Authorization: Bearer {api_key}`, JSON body: `VoyageRequest { input: batch, model: self.model.clone(), input_type: Some("document".to_string()), truncation: true }`
    - On success (200): parse VoyageResponse, sort embeddings by index, return EmbeddingResult
    - On 401/403: return `LocalIndexError::Embedding("Authentication failed: invalid VOYAGE_API_KEY")` immediately (no retry)
    - On 429/5xx: increment attempt, compute delay = min(BASE_DELAY_MS * 2^(attempt-1) + random_jitter, MAX_DELAY_MS), `tracing::warn!` with attempt/max/delay, `tokio::time::sleep(delay)`, retry
    - On max retries exceeded: return `LocalIndexError::Embedding(format!("API request failed after {MAX_RETRIES} retries: {status}"))`

    6. **Unit tests** (use wiremock for HTTP mocking):
    - `test_embed_success`: Mock 200 response with valid VoyageResponse JSON, verify returned embeddings match
    - `test_embed_auth_failure`: Mock 401 response, verify immediate Embedding error (no retry)
    - `test_embed_retry_on_429`: Mock 429 then 200, verify success on 2nd attempt
    - `test_embed_retry_exhausted`: Mock 500 for all attempts, verify error after MAX_RETRIES
    - `test_embed_batching`: Send 120 texts, verify wiremock receives 3 requests (batches of 50, 50, 20)
    - `test_model_id`: Verify model_id() returns "voyage-3.5"
    - `test_dimensions`: Verify dimensions() returns 1024
