Implement `pub async fn ingest_asset_path(
  vault: &Path,
  asset_rel: &Path,
  data_dir: &Path,
  max_bytes: usize,
  max_pdf_pages: usize,
  client: Option<&AnthropicAssetClient>,
) -> Result<ChunkedFile, LocalIndexError>` in `ingest.rs`:

**PRE-14:** If the code path requires Anthropic (NeedsVision PDF or raster image) and `client` is `None`, return `LocalIndexError::Credential` with message containing `ANTHROPIC_API_KEY` (do not WARN-skip).

1. Resolve absolute path = `vault.join(asset_rel)`; verify `strip_prefix(vault)` roundtrip.
2. Read file capped at `max_bytes` (error with `asset too large` if exceeded).
3. Match extension:
   - **`.pdf`**: classify; if `TextFirst`, `extract_text_pdf_as_markdown`; build synthetic markdown string with top matter comment line `# Source: <filename>` (literal prefix) + body; call `chunk_markdown(&synthetic, asset_rel)` so **every chunk’s `file_path` equals `asset_rel`**.
   - **`NeedsVision`**: require `client.is_some()`; call `rasterize_pdf_pages_to_png(bytes, max_pages)` from `pdf_raster.rs` (Plan 09-02); for each page PNG, call `client.describe_raster_page(&png)`; concatenate page descriptions with `\n\n---\n\n` between pages into synthetic markdown; then `chunk_markdown` with `asset_rel` as path. `max_pages` from `LOCAL_INDEX_MAX_PDF_PAGES` (default **30**). If rasterization returns zero pages, return error containing substring `no pages rasterized`.
   - **Image** (`png|jpg|jpeg|webp`): require `client`; call `describe_image` with correct media type; synthetic markdown = `# <filename>\n\n` + description; `chunk_markdown`.
4. Optionally write synthetic body to cache file under `cache_path_for_hash` using SHA-256 of source bytes (use `sha2` crate already in workspace) — **for debugging/retry only**; must not change `Chunk.file_path`.

Export `ingest_asset_path` from `assets/mod.rs`.
