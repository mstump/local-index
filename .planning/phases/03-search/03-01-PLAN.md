Struct definition:
```rust
pub struct SearchEngine<'a> {
    store: &'a ChunkStore,
    embedder: &'a dyn Embedder,
}
```

Implement these methods:

1. **`pub fn new(store: &'a ChunkStore, embedder: &'a dyn Embedder) -> Self`**

2. **`pub async fn search(&self, opts: &SearchOptions) -> Result<SearchResponse, LocalIndexError>`** -- Dispatches to `semantic_search`, `fts_search`, or `hybrid_search` based on `opts.mode`. After getting results, applies `min_score` filter (filtering by `similarity_score >= min_score`). Then if `opts.context > 0`, calls `fetch_context_chunks`. Returns wrapped SearchResponse per D-02.

3. **`pub async fn ensure_fts_index(&self) -> Result<(), LocalIndexError>`** -- Calls `self.store.table().create_index(&["body"], Index::FTS(FtsIndexBuilder::default())).execute().await`. Maps error to `LocalIndexError::Database`.

4. **`async fn semantic_search(&self, opts: &SearchOptions) -> Result<Vec<SearchResult>, LocalIndexError>`**:
   - Embed query: `self.embedder.embed(&[opts.query.clone()]).await?` then get first vector
   - Build query: `self.store.table().query().nearest_to(&query_vec)?.distance_type(DistanceType::Cosine).limit(effective_limit)`
   - If `path_filter` set: `.only_if(format!("file_path LIKE '{}%'", path_filter.replace("'", "''")))` -- escape single quotes
   - Execute, collect RecordBatches via `.try_collect::<Vec<RecordBatch>>().await`
   - Extract results using `extract_results_from_batches` helper
   - For semantic mode: `similarity_score = semantic_score = 1.0 - (distance / 2.0)`, `fts_score = None`
   - If `tag_filter` set: post-filter results by deserializing frontmatter JSON and checking `tags` array contains the filter value
   - Truncate to `opts.limit` after filtering

5. **`async fn fts_search(&self, opts: &SearchOptions) -> Result<Vec<SearchResult>, LocalIndexError>`**:
   - Call `self.ensure_fts_index().await?` first
   - Build query: `self.store.table().query().full_text_search(FullTextSearchQuery::new(opts.query.clone())).limit(effective_limit)`
   - If `path_filter`: add `.only_if(...)` same as semantic
   - Execute and collect
   - Extract results; `_score` column gives BM25 score. Normalize: find max `_score` in results, divide all by max (if max == 0.0, set all to 0.0)
   - `similarity_score = fts_score = normalized_score`, `semantic_score = None`
   - Post-filter by tag if needed, truncate to limit

6. **`async fn hybrid_search(&self, opts: &SearchOptions) -> Result<Vec<SearchResult>, LocalIndexError>`**:
   - Call `self.ensure_fts_index().await?`
   - Embed query for vector component
   - Build hybrid: `self.store.table().query().full_text_search(FullTextSearchQuery::new(opts.query.clone())).nearest_to(&query_vec)?.distance_type(DistanceType::Cosine).rerank(Arc::new(RRFReranker::new(60.0))).limit(effective_limit)`
   - If `path_filter`: `.only_if(...)`
   - Execute and collect
   - Extract results with all three score columns: `_distance`, `_score`, `_relevance_score`
   - Normalize: `semantic_score = 1.0 - (distance / 2.0)`, `fts_score = score / max_score`, `similarity_score = relevance_score / max_relevance_score`
   - Post-filter by tag, truncate

7. **`async fn fetch_context_chunks(&self, results: &mut Vec<SearchResult>, context_n: usize) -> Result<(), LocalIndexError>`**:
   - For each result, query same file's chunks: `self.store.table().query().only_if(format!("file_path = '{}'", result.file_path.replace("'", "''")))` with Select::Columns for file_path, heading_breadcrumb, body, line_start, line_end, frontmatter_json
   - Sort by line_start
   - Find matching chunk by line_range
   - Take N before and N after
   - Add context chunks to results with `is_context: Some(true)`, `context_for_index: Some(original_index)`, `similarity_score: 0.0`, other scores None

8. **Helper: `fn extract_results_from_batches(batches: &[RecordBatch], score_mode: ScoreMode) -> Vec<RawResult>`** where `RawResult` is an internal struct holding the raw Arrow column values before score normalization. Use `column_by_name` + `downcast_ref::<StringArray>()` for string columns, `downcast_ref::<UInt32Array>()` for line numbers. For float score columns (`_distance`, `_score`, `_relevance_score`), try `Float32Array` first, then `Float64Array` fallback per Pitfall 4 in RESEARCH.md. If a score column is missing (e.g., `_distance` in FTS-only mode), skip it.

**Effective limit calculation for tag filtering:** When `tag_filter` is set, multiply the requested limit by 3 to over-fetch, then post-filter in Rust. Accept that results may be fewer than requested for rare tags (v1 simplification per RESEARCH.md Pitfall 6).

**Error handling:** Wrap all LanceDB errors in `LocalIndexError::Database(e.to_string())`. If table is empty or query returns no results, return empty `Vec<SearchResult>` (not an error per CONTEXT.md specifics).

Add unit tests in engine.rs `#[cfg(test)] mod tests`:
- `test_score_normalization_semantic`: Given `_distance = 0.5`, assert `semantic_score = 1.0 - (0.5 / 2.0) = 0.75`
- `test_score_normalization_fts`: Given scores `[3.0, 6.0, 9.0]`, assert normalized to `[0.333, 0.666, 1.0]`

- `test_tag_filter_logic`: Given a frontmatter JSON `{"tags":["rust","search"]}`, assert tag "rust" matches and tag "python" does not
