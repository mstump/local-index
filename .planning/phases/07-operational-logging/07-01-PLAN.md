LOG-03: Suppress LanceDB/Lance internal tracing below WARN by adding `lancedb=warn,lance=warn` to the default EnvFilter.

In `src/main.rs`, modify the `init_logging` function (line ~15-28):

Change the filter construction from:
```rust
let filter = EnvFilter::try_from_default_env()
    .unwrap_or_else(|_| EnvFilter::new(log_level));
```

To:
```rust
let filter = EnvFilter::try_from_default_env().unwrap_or_else(|_| {
    EnvFilter::new(format!("{},lancedb=warn,lance=warn", log_level))
});
```

This ensures:
- When RUST_LOG is NOT set: the default filter is `info,lancedb=warn,lance=warn` (or whatever --log-level is), suppressing noisy LanceDB messages.
- When RUST_LOG IS set: the user's explicit filter takes full precedence (they can set `RUST_LOG=debug,lancedb=debug` to restore LanceDB output).

Do NOT modify the RUST_LOG precedence path -- only the fallback default.
